# Result Interpretation

## Pixel-Level Segmentation Metrics

For binary foreground masks:

- **Intersection over Union (IoU)** is the intersection divided by the union of predicted and labeled foreground pixels.
- **Precision** is the fraction of predicted foreground pixels that match labeled foreground.
- **Recall** is the fraction of labeled foreground pixels recovered by the prediction.
- **F1** is the harmonic mean of precision and recall.

These metrics describe pixel agreement with a defined mask. They do not directly measure object recognition, downstream utility, runtime, or robustness outside the tested conditions.

## Edge Metrics

The edge experiment uses precision, recall, and F1 with a two-pixel positional tolerance. A predicted edge counts toward precision when it falls near a reference edge; a reference edge counts toward recall when a prediction falls near it.

This dilation-based policy avoids penalizing minor rasterization shifts, but it does not enforce one-to-one boundary matching. Edge F1 is therefore not interchangeable with the segmentation F1 reported elsewhere.

## Stability Metrics on Unlabeled Photographs

The public denoising and edge samples compare noisy-image outputs with outputs from the clean source image. These values measure algorithmic stability under injected noise.

The clean output is generated by an algorithm, not by a human annotation. A high stability score can preserve a systematic error and must not be described as semantic accuracy.

## Foreground Fraction

Foreground fraction is the proportion of pixels selected by a thresholding configuration. It is useful for showing that parameters materially change an output. Without a labeled mask or task target, a larger or smaller fraction is not inherently better.

## Cross-Experiment Summary

`results/experiment_summary.csv` selects one reviewable value from each experiment. Every row records its metric and evidence scope.

The rows must not be ranked because:

- some use IoU while the edge experiment uses tolerance-aware F1;
- some values describe one controlled condition while others are means;
- synthetic ground truth and public trimaps represent different data distributions;
- each experiment answers a different research question.

Use the summary to locate evidence. Use the detailed CSV, comparison image, interpretation, and limitations to evaluate a conclusion.

## Examples of Supported Conclusions

Supported:

> Under the generated salt-and-pepper condition, median filtering raises Canny edge F1 from 0.151 without filtering to 1.000 with the fixed default parameters.

Not supported:

> Median filtering always solves noisy edge detection.

Supported:

> Fixed-inset GrabCut has a mean IoU of 0.745 on the selected six-image Oxford-IIIT Pet subset and outperforms both Otsu polarity baselines on those images.

Not supported:

> GrabCut achieves 0.745 IoU on the Oxford-IIIT Pet Dataset.

The narrower statements preserve the tested condition, comparison, parameters, and sample scope.
